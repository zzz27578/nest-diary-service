# 小窝日记双项目实施计划

> 本计划用于审批和后续执行。实施时按任务清单逐项推进，先打底层，再做页面，再做部署包装。

## 一、核心目标

我们要做的不是普通博客，也不是让 bot 模仿人类去登录后台写文章。

我们要做的是一套 **Bot 原生的小窝日记系统**：

- Bot 有自己的小窝本体，用来存日记、图片、附件、归档和搜索索引。
- AstrBot 通过插件连接这个小窝，给 bot 提供“写、搜、读、改、归档、上传媒体”等工具。
- 人类可以通过密码登录网页后台查看和管理，但网页不是 bot 的主要操作入口。
- 日记要有 bot 自己的评价、情绪和长期记忆线索，不能只是聊天流水账。
- 数据必须长期可靠、可迁移、可备份、可追溯。

最终效果：

```text
Bot 想写日记：
  调用写入工具 -> 小窝保存 Markdown -> 更新索引 -> 记录修订历史

Bot 想回忆：
  调用搜索工具 -> 小窝用索引找相关内容 -> 只返回命中的片段和必要原文

你想查看：
  打开小窝网站 -> 输入管理员密码 -> 浏览日记、图片、归档、搜索结果
```

## 二、项目拆分

当前工作区先建立两个子文件夹，未来可以拆成两个独立仓库。

```text
C:\Users\29505\Desktop\记忆插件\
  astrbot-plugin-nest-diary-connector\
    AstrBot 标准插件，负责连接小窝本体。

  nest-diary-service\
    小窝本体服务，负责网站、API、存储、搜索、归档和部署。
```

### 1. AstrBot 插件项目

目录：

```text
astrbot-plugin-nest-diary-connector/
```

职责：

- 按标准 AstrBot 插件形式安装。
- 保存小窝服务地址和 bot 专属 token。
- 提供内置 skills。
- 封装 bot 可直接使用的工具。
- 把 bot 的操作请求转发给小窝服务。
- 不保存大量日记和图片。

一句话：

```text
插件是 bot 的门牌号和工具箱。
```

### 2. 小窝本体服务项目

目录：

```text
nest-diary-service/
```

职责：

- 提供私密网页后台。
- 提供 bot 专属 API。
- 存储 Markdown 日记。
- 存储图片、语音、附件。
- 维护 SQLite 元数据和全文搜索索引。
- 记录每次修改的版本历史。
- 自动归档日记。
- 支持旧 `daily_diary.txt` 导入。
- 提供普通 Docker Compose 部署。
- 额外提供 1Panel 本地应用包，但不强制依赖 1Panel。

一句话：

```text
小窝服务是 bot 真正住的地方。
```

## 三、硬性原则

这些原则后续实现时不能偏离。

### 1. Bot 不操作网页

Bot 不登录网页，不点按钮，不填表单。

Bot 只使用工具：

```text
write_diary
search_diary
read_diary
revise_diary
archive_diary
upload_media
rollback_revision
```

网页只是给人类查看和管理。

### 2. 插件保持轻量

AstrBot 插件不保存大体量数据。

插件只保存：

```text
小窝服务地址
bot token
少量配置
skills
工具封装
```

日记、图片、索引、归档全部放在 `nest-diary-service`。

### 3. 小窝服务独立可运行

小窝服务必须可以不依赖 AstrBot 单独启动。

最低要求：

```text
docker compose up -d
```

就能启动网站和 API。

1Panel 只是额外部署方式，不是唯一方式。

### 4. 双入口认证

Bot 入口和网页入口分开。

```text
Bot API:
  使用 NEST_BOT_API_TOKEN

网页后台:
  使用 NEST_ADMIN_PASSWORD
```

这样网页密码泄露不会等于 bot 工具权限泄露，bot token 泄露也不会直接登录网页。

### 5. 不内置人设

插件和服务不写死“小莫”的人格词。

人设仍然由 AstrBot 原生人格系统负责。

本项目只提供：

```text
工具
存储
索引
归档
写入规范
skills
```

### 6. 日记不是流水账

写入规范必须要求日记包含：

```text
发生了什么
bot 的主观评价
bot 的情绪
和老爸有关的长期记忆
后续线索
关联材料
```

### 7. 禁止默认全量读取

Bot 想回忆时，必须先走索引。

查找顺序：

```text
明确日期 -> 读取当天日记
关键词 -> SQLite FTS 全文搜索
人物/标签/事件 -> 元数据索引
模糊回忆 -> 未来可选向量搜索
```

不允许一上来读取所有日记文件。

### 8. 所有改动可追溯

Bot 完全放权，但不能无痕修改。

每次写入、修改、归档、删除前后都要记录：

```text
时间
操作工具
操作原因
目标对象
修改前快照
修改后结果
是否可回滚
```

### 9. 数据保持可读可迁移

日记正文以 Markdown 文件作为真实数据。

SQLite 只做索引和管理，不作为唯一真相。

这样即使服务坏了，也能直接打开文件看日记。

## 四、底层存储方案

### 1. 日记文本

按年月日存放：

```text
nest-diary-service/data/
  diary/
    2026/
      05/
        2026-05-13.md
```

日记文件格式：

```markdown
---
id: diary_2026_05_13
date: 2026-05-13
timezone: Asia/Shanghai
mood: ["认真", "有点烦"]
tags: ["小窝", "AstrBot"]
people: ["老爸", "小莫"]
importance: 4
source: bot
revision: 1
created_at: 2026-05-13T03:00:00+08:00
updated_at: 2026-05-13T03:00:00+08:00
---

# 2026-05-13

## 今天发生了什么

## 我的评价

## 我的情绪

## 关于老爸的记忆

## 后续线索

## 关联材料
```

### 2. 图片、语音、附件

媒体文件使用 SHA256 内容寻址，避免重复和改名混乱。

```text
media/
  blobs/
    sha256/
      ab/
        cd/
          abcd1234....png
  variants/
    ab/
      cd/
        abcd1234.thumb.webp
  by-date/
    2026/
      05/
        2026-05-13/
          manifest.json
```

真实文件只存一份，日期目录用 `manifest.json` 记录关联关系。

### 3. SQLite 索引

SQLite 负责：

```text
日记元数据
标签、人物、事件
全文搜索 FTS5
媒体清单
修订历史
操作日志
```

计划数据库：

```text
data/index/nest.sqlite
```

### 4. 未来扩展

底层预留但第一版不强制实现：

```text
sqlite-vec / 向量搜索
MinIO / S3 媒体存储
Litestream / restic 备份
多用户或多 bot 小窝
```

## 五、两个项目的连接方式

### 1. 插件配置

AstrBot 插件里配置：

```text
service_url = http://nest-diary:28080
bot_api_token = 一串长 token
request_timeout_seconds = 30
daily_write_enabled = true
notify_after_write = true
```

### 2. 服务配置

小窝服务里配置：

```text
NEST_BOT_API_TOKEN=同一串长 token
NEST_ADMIN_PASSWORD=网页登录密码
NEST_DATA_DIR=/app/data
TZ=Asia/Shanghai
```

### 3. 调用方式

插件调用小窝服务：

```text
GET  /api/v1/status
POST /api/v1/diary/write
GET  /api/v1/diary/search
GET  /api/v1/diary/{date}
POST /api/v1/diary/revise
POST /api/v1/archive/run
POST /api/v1/media/upload
GET  /api/v1/skills
POST /api/v1/skills/reload
```

每次修改类操作都带：

```json
{
  "actor": "astrbot",
  "intent": "daily_diary_write",
  "reason": "凌晨定时写入日记",
  "idempotency_key": "2026-05-13-daily-write"
}
```
