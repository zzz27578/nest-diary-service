# 小窝日记使用手册

## 1. 部署小窝服务

复制环境变量文件：

```bash
cp .env.example .env
```

编辑 `.env`：

```text
NEST_PORT=28080
NEST_ADMIN_PASSWORD=你的网页登录密码
NEST_BOT_API_TOKEN=一串很长的 bot token
TZ=Asia/Shanghai
```

启动：

```bash
docker compose up -d
```

浏览器打开：

```text
http://服务器IP:28080
```

输入 `NEST_ADMIN_PASSWORD` 登录。

## 2. 配置 AstrBot 插件

安装 `astrbot-plugin-nest-diary-connector` 后，在插件配置里填：

```text
service_url=http://nest-diary:28080
bot_api_token=和 NEST_BOT_API_TOKEN 相同
```

如果 AstrBot 和小窝服务不在同一个 Docker 网络，把 `service_url` 改成可访问地址，例如：

```text
http://服务器IP:28080
```

## 3. 检查连接

在 AstrBot 里发送：

```text
/小窝状态
```

正常会回复小窝在线。

## 4. 数据位置

小窝服务数据在：

```text
nest-diary-service/data/
```

核心目录：

```text
diary/       Markdown 日记
media/       图片、语音、附件
index/       SQLite 搜索索引
revisions/   修改前快照
archive/     月度/年度归档
```

## 5. 1Panel 安装

1Panel 本地应用包在：

```text
deploy/1panel/nest-diary/
```

把该目录复制到 1Panel 本地应用目录后刷新本地应用列表即可。  
1Panel 只是可选部署方式，不影响普通 Docker Compose 部署。
